About Me

I am a second year Games Design student at the university Of Staffordshire, focusing on gameplay design, system design, and core system iteration. The work I do focuses on designing, testing, and iterating core gameplay systems.
All of my showcased projects were completed independly as a part of a university assignments, structured in a way to simulate production constraints such as prototyping, implementation limits and testing iteration.

Projects

A few projects that showcase my skills and design.

Unity – Core Loop Prototyping

Role: Gameplay Designer (Solo) · Platform: PC

This university assignment was designed to simulate real and professional prototyping. with a focus on time-boxing, scope managment and rapid iteration.
I was resposible for defining the core gameplay idea, building a prototype, having the prototype tested and iteratation within a fixed timeframe.

  • Design focus: Prototyping, Testing, Iteration
  • Constraints: Fixed development time, no visual machine.
  • Results: Conducted informal playtests, Observed player engagement and pacing

Unity · Visual Studio · Github · C# · Github

Unreal Engine – Gameplay Systems implementation

Role: Gameplay Designer (Solo) · Focus: Gameplay Systems, implementation

This project was focused on how systems can be made using both C++ and Blueprints inside Unreal Engine.
The goal behind this was to connect gameplay design intent with technical implementation enabling better control over the player experience.

  • Design focus: Gameplay systems structure, iteratation through implementation
  • Designed systems: Player abilities and input handling
  • Implementation: Core logic implementated through C++, blueprints used for final iteratation and tweaks

Unreal Engine · Rider · C++ · Blueprints · Github
